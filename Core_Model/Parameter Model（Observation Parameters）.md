# Thought Algorithm Skeleton Theory

## Parameter Model (Observation Parameters) Ver.1

---

# Overview

This document defines the **Observation Parameter Model** of the Thought Algorithm Skeleton Theory, also referred to as the **Skeleton Theory**.

The purpose of this model is not to treat human cognition as a fixed personality classification.

Instead, it focuses on observing tendencies in the processes through which cognitive structures are:

* Updated
* Maintained
* Reconstructed

in response to incoming information and changing conditions.

The parameters defined in this model are **not intended for**:

* Ability assessment
* Superiority or inferiority judgments
* Personality classification

The primary subject of observation is the tendency of cognitive algorithms and their processing patterns.

---

# 1. Basic Parameters

## 1.1 Update Pace (UP)

### Definition

The tendency regarding the speed at which structural updating begins when new or contradictory information is received in relation to an existing cognitive structure.

### Observation Targets

* Speed of hypothesis revision
* Adaptation to new information
* Initiation of updates after detecting differences
* Frequency of updates

### Tendency Examples

**Higher tendency:**

* More readily incorporates new information
* Revises hypotheses quickly
* Adapts quickly to environmental changes

**Lower tendency:**

* More likely to maintain existing structures
* Places greater emphasis on consistency
* Requires sufficient evidence before updating

### Related Parameters

SR / RF / CT

---

## 1.2 Convergence Threshold (CT)

### Definition

The tendency regarding the point at which a person determines that a situation or problem has been sufficiently understood for the current purpose during information exploration and cognitive processing.

### Observation Targets

* Timing of exploration termination
* Speed of hypothesis formation
* Maintenance of uncertainty

### Tendency Examples

**Higher tendency:**

* Conducts extensive information exploration
* Delays finalizing hypotheses
* Maintains provisional or uncertain states

**Lower tendency:**

* Forms hypotheses at an earlier stage
* Makes use of provisional judgments
* More readily transitions from analysis to action

### Related Parameters

UP / MC

---

## 1.3 Structural Retention (SR)

### Definition

The tendency to maintain an existing cognitive structure while updating only the parts that require modification.

### Observation Targets

* Long-term consistency
* Stability after updating
* Ability to make localized modifications

### Tendency Examples

**Higher tendency:**

* Maintains structural stability
* More readily performs partial modifications

**Lower tendency:**

* More readily performs large-scale reconstruction
* Allows cognitive structures to remain more fluid

### Related Parameters

UP / RF

---

## 1.4 Reconstruction Flexibility (RF)

### Definition

The tendency to reorganize an existing cognitive structure and reconstruct it into a new structure.

### Observation Targets

* Integration across different domains
* Shifts in perspective
* Model redesign

### Tendency Examples

**Higher tendency:**

* More readily forms new structures
* More readily changes existing frameworks

**Lower tendency:**

* Prioritizes existing structures
* Performs structural redesign less frequently

### Related Parameters

SR / AL

---

## 1.5 Abstraction Level Preference (AL)

### Definition

The tendency regarding which levels of abstraction are preferentially used during information processing.

### Example Hierarchy

```text
Concrete
    ↓
Example
    ↓
Pattern
    ↓
Structure
    ↓
Principle
    ↓
Meta-Principle
```

### Observation Targets

* Extraction of common principles
* Frequency of abstraction
* Selection of cognitive hierarchy

### Related Parameters

RF

# 2. Cognitive Resource Parameters

## 2.1 Automation (AU)

### Definition

The tendency for cognitive processing to be performed without conscious control.

### Observation Targets

* Habitual processing
* Skilled processing
* Automatic or implicit processing

---

## 2.2 Semi-Automation (SA)

### Definition

The tendency for conscious control to intervene in normally automated processing when circumstances or environmental conditions change.

### Observation Targets

* Response to changes in circumstances
* Switching from automatic to conscious processing

### Related Parameters

AU

---

## 2.3 Resource Allocation (RA)

### Definition

The tendency regarding how available cognitive resources are allocated among different targets.

### Observation Targets

* Allocation of attention
* Prioritization
* Response to changes in circumstances

### Example Allocation Targets

* Observation
* Understanding others
* Problem solving
* Meaning generation
* Future prediction

### Related Parameters

PP

---

## 2.4 Parallel Processing (PP)

### Definition

The tendency to maintain and process multiple cognitive processes simultaneously.

### Observation Targets

* Simultaneous processing of multiple information streams
* Monitoring multiple states during conversation
* Processing across multiple tasks

### Related Parameters

RA

---

# 3. Interpretation Principles

## 3.1 No Single Parameter Evaluation

An individual should not be evaluated based on a single parameter.

Cognitive processing tendencies should be interpreted as the result of interactions among multiple parameters.

---

## 3.2 Dynamic Processing Model

The parameters are not fixed personality attributes.

They should be treated as tendencies in cognitive processing that may change depending on:

* Environment
* Goals
* Experience
* Cognitive state

---

## 3.3 No Superiority Ranking

Expressions such as "higher" and "lower" describe the direction of a processing tendency.

They do not represent performance evaluation, superiority, or inferiority.

A higher or lower tendency may be advantageous or disadvantageous depending on the environment, purpose, and context in which the cognitive process occurs.

* Human classification

* Superiority or inferiority judgments

The model should not be used to define individuals through fixed categories.

---

# Summary

The Observation Parameter Model provides a framework for describing tendencies in how cognitive structures are updated, maintained, reconstructed, and processed.

The parameters are intended to support the observation and understanding of dynamic cognitive processes rather than the classification or evaluation of individuals.

The model assumes that cognitive processing is dynamic and context-dependent.

Therefore, the same individual may exhibit different processing tendencies under different environmental conditions, goals, experiences, and cognitive states.

> **The purpose of observation is not to classify people, but to better understand how cognitive structures change.**
